================================================================================
                        CONFIGURATION GUIDE
================================================================================

Edit js/config.js to customize:

1. Product & Chapter Names
   productName: 'Faulty Link'
   chapterName: 'Den'
   chapterCityLabel: 'Denver'

2. Navigation Links (single source)
   navLinks: [
     { href: '/', label: 'Home' },
     { href: '/start/', label: 'Start' },
     // Add/remove as needed
   ]

3. Base Path (for GitHub Pages project sites)
   basePath: ''              // root domain or Cloudflare Pages
   basePath: '/repo-name'    // GitHub Pages project site

4. Form Fields
   fields: [
     { key: 'name', label: 'Name', type: 'text', required: true },
     { key: 'email', label: 'Email', type: 'email', required: true },
     // Add more: text, email, textarea, select
   ]

5. Submission Mode
   submitMode: 'clipboard'   // safe, default (no backend)
   submitMode: 'mailto'      // opens email client
   submitMode: 'postJson'    // POSTs to your endpoint

6. postJson Endpoint (for postJson mode)
   endpoint: 'https://webhook.site/your-unique-id'  // test endpoint
   endpoint: 'https://your-api.example.com/signups'  // production

================================================================================
                        POSTJSON CORS TESTING
================================================================================

To test postJson without building a backend:

1. Visit https://webhook.site
2. Copy the unique URL (e.g., https://webhook.site/abc-123-def)
3. In config.js:
   submitMode: 'postJson'
   endpoint: 'https://webhook.site/abc-123-def'
4. Reload page, fill form, click Submit
5. Success message: "Submitted. Thank you!"
6. Check webhook.site to see the POST request with your form data

Note: Your endpoint must support CORS and respond with 2xx for success.
